# AI Editor MVP 后凭据加密待办

更新时间：2026-07-15
状态：**已确认延期，MVP 完成后实施**

## 1. 当前产品决策

当前尚未部署中央服务器。账号服务和中央模型网关在调试阶段先运行于本机，并使用
SQLite。

为了优先跑通 MVP，暂不实现上游凭据的信封加密：

- ChatGPT 上游账号凭据、OpenAI/DeepSeek API Key 和 Relay 密钥在调试数据库中暂时以
  明文保存；
- 该实现只允许用于本机开发和 MVP 功能验证；
- 在中央服务对外部署、接入真实用户或保存生产上游凭据前，必须完成本文档中的加密
  待办。

本决策不代表明文凭据可以进入 Git、安装包、日志、测试报告或公开备份。

## 2. 不能延期的基础安全功能

以下项目属于账号系统基础功能，MVP 阶段仍必须实现：

- 产品用户密码必须使用带随机盐的安全密码哈希，禁止保存可还原的明文密码；
- 邀请码、一次性授权码和临时密码只保存哈希或不可逆摘要；
- 服务端 Refresh Token 只保存哈希，并启用轮换、撤销和重放检测；
- Windows 客户端使用 Credential Manager/DPAPI，macOS 客户端使用 Keychain 保存
  Refresh Token；
- Access Token 只保存在 Code/Proxy 进程内存；
- 管理 API 只返回脱敏后的账号和密钥信息；
- 日志、错误、诊断、调用审计和导出报告不得包含密码、Token、API Key 或完整上游
  凭据；
- SQLite 数据库、开发密钥、运行数据和备份目录必须被 Git 与产品安装包排除；
- 本机调试服务默认只监听 `127.0.0.1`，不得静默监听公网地址。

## 3. MVP 明文凭据的临时边界

MVP 实现需要遵守以下边界，降低后续迁移成本：

1. 所有上游凭据只能通过统一的 `CredentialRepository`/凭据存储模块读写，业务路由
   不得直接读取数据库字段。
2. 凭据记录预留存储格式和密钥版本字段，例如：

   ```text
   storage_format = plaintext-v1
   key_version = null
   ```

3. 管理页面保存凭据后不得再次显示完整值，只显示固定长度的脱敏预览。
4. 本机数据库文件使用当前操作系统用户可读写的最小文件权限。
5. 禁止自动备份包含明文上游凭据的数据库；确需手动备份时必须显示风险提示。
6. 自动化测试只能使用无效的测试密钥，不能复制开发者真实账号或 API Key。
7. 服务启动日志必须明确标记：

   ```text
   Development credential storage: plaintext-v1
   Not approved for a public or production deployment.
   ```

## 4. MVP 后必须实现的信封加密

### 4.1 加密格式

- 每条上游凭据生成独立随机数据密钥（DEK）；
- 使用 AES-256-GCM 加密凭据正文；
- 每次加密使用独立随机 nonce；
- 将记录 ID、组织 ID、凭据类型和版本作为附加认证数据；
- 数据库保存密文、nonce、认证标签、被包装的数据密钥和密钥版本；
- 主密钥只用于包装 DEK，不直接批量加密业务数据。

### 4.2 主密钥管理

- 本机迁移测试可使用被 Git 忽略的独立开发密钥文件；
- 正式中央服务必须从部署 Secret/KMS/HSM 获取主密钥；
- 主密钥不得写入数据库、源码、镜像、安装包或普通环境配置文件；
- 支持新写入使用新密钥版本、旧记录按批次重新包装；
- 密钥轮换期间服务不中断。

### 4.3 明文到密文迁移

- 提供可重复执行、可恢复的迁移命令；
- 迁移前验证主密钥和数据库备份策略；
- 逐条读取 `plaintext-v1`，加密后回读验证；
- 只有验证成功后才删除原明文字段；
- 迁移失败不得留下“部分明文、部分密文但无法识别”的记录；
- 迁移完成后扫描数据库、日志、临时文件和备份，确认不存在明文凭据；
- 提供迁移统计，但不得在输出中显示凭据内容。

### 4.4 权限与审计

- 只有中央模型网关的凭据模块可以请求解密；
- 普通用户和二级管理员不能读取密文、密钥版本或底层凭据结构；
- 一级管理员可以替换或撤销凭据，但不能通过管理 API 取回完整明文；
- 每次新增、替换、使用、迁移和删除凭据都写入不含秘密正文的审计日志。

## 5. 完成验收标准

中央服务公开部署前，以下检查必须全部通过：

- 数据库和数据库备份中搜索不到测试 API Key 明文；
- 管理 API、错误响应、日志和诊断报告中搜索不到测试 API Key 明文；
- 使用错误主密钥时服务拒绝启动或拒绝路由，不能返回损坏数据；
- 篡改密文、nonce、认证标签或附加认证数据后解密必须失败；
- 主密钥轮换后新旧凭据均可使用；
- 明文迁移命令可中断恢复并保持幂等；
- Windows/macOS 客户端令牌安全存储验证通过；
- 安装包与发布资源扫描不包含数据库、开发主密钥或真实凭据。

## 6. 投入评估

只实现本机 AES 加密并不等于完成生产安全闭环。正式实现还包括 KMS、密钥轮换、迁移、
权限、审计、备份和故障恢复。

当前粗略评估：

- 本机加密格式与凭据存储模块：约 2–4 个开发日；
- KMS/部署密钥、轮换与迁移：约 3–7 个开发日；
- 安全测试、备份恢复和发布验收：约 3–5 个开发日。

具体投入需要在正式部署平台和 KMS 供应商确定后重新评估。

## 7. 发布阻断条件

出现以下任一情况时，不得把账号服务或中央模型网关提升为公开生产服务：

- 仍存在 `storage_format = plaintext-v1` 的真实上游凭据；
- 正式服务使用本机开发密钥文件；
- 数据库备份包含明文凭据；
- 管理 API 可以返回完整上游凭据；
- 日志、诊断或调用审计可能记录秘密正文；
- 尚未完成密钥丢失、轮换失败和迁移中断的恢复演练。
